AJAX using jQuery vs ES6 fetch()

Doing AJAX using jQuery is much simpler than using the standard JavaScript XmlHttpRequest. With ES6, there's a new method available via plain JavaScript - fetch().

Like jQuery, fetch() uses Promises to handle asynchronous execution, rather than simple callbacks. ES6 promises are implemented a little differently from jQuery's, but the principles are the same in each.

Documentation:

  • jQuery getJSON() with Promises
  • MDN Using Fetch

Notice that fetch() returns a Promise. The function called in its .then() method is provided a Response object as its argument. The Response has methods like json(), which itself returns a Promise, meaning you can chain on another .then() which will run when the json is parsed.

All of this is asynchronous - .then() only runs when its Promise either completes successfully (fulfilled) or fails (rejected).
